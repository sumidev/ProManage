# Project: **Interview Prep Dashboard**

(kyunki tum job switch kar rahe ho — ye project tumhare profile ko directly strong karega)

## Short description

Ek single-page dashboard jisme tum apne interview tasks, topic-wise questions, mock interviews, and company applications manage karoge. Crucial features: auth, CRUD, filters, pagination, optimistic updates, tests, deploy.

---

# MVP Features (must-have)

1. **Authentication** (email + password, mock JWT)
2. **Dashboard** — summary cards: total applications, upcoming mocks, topics to revise.
3. **Applications CRUD** — Add company applications (company, role, stage, appliedOn, expectedSalary).
4. **Topics CRUD** — Topics (e.g., React, Redux, System Design) with progress% and notes.
5. **Questions CRUD** — add questions under topics, mark as solved/unsolved, difficulty tag.
6. **Mock Interview Scheduler** — date/time, interviewer, status. Calendar list view.
7. **Filters & Search** — filter applications by stage, search questions by text/tag.
8. **Optimistic UI updates** for marking solved, and proper error rollback.
9. **Persistent storage** via a simple backend (JSON Server / Express + lowdb) or localStorage for prototype.
10. **Responsive UI** and deployable to Vercel/Netlify + backend to Render/Heroku (or keep backend mocked).

# Redux structure (recommended)

* `store/` setup with Redux Toolkit `configureStore`.
* `slices/`:

  * `authSlice` (login/logout, token)
  * `applicationsSlice` (list, add, update, delete)
  * `topicsSlice`
  * `questionsSlice` (with selectors for solved/unsolved)
  * `uiSlice` (snackbar, loading states)
* Use thunks or RTK Query for async API calls.

### Minimal store setup (example)

```js
import { configureStore } from '@reduxjs/toolkit'
import authReducer from './slices/authSlice'
import appsReducer from './slices/applicationsSlice'
export const store = configureStore({
  reducer: {
    auth: authReducer,
    apps: appsReducer,
    // ...
  },
})
```

# Quick starter code snippet (Question solved toggle thunk)

```js
// slices/questionsSlice.js (RTK)
import { createSlice, createAsyncThunk } from '@reduxjs/toolkit'
export const toggleSolved = createAsyncThunk(
  'questions/toggleSolved',
  async ({ id, solved }, { rejectWithValue, dispatch }) => {
    // optimistic update simulated by client; call API then confirm
    try {
      const res = await fetch(`/api/questions/${id}`, {
        method: 'PATCH',
        headers:{'Content-Type':'application/json'},
        body: JSON.stringify({ solved })
      })
      if(!res.ok) throw new Error('Server error')
      return await res.json()
    } catch (err) {
      return rejectWithValue({ id, error: err.message })
    }
  }
)
const questionsSlice = createSlice({
  name: 'questions',
  initialState: { items: [], status: 'idle', error: null },
  reducers: {
    optimisticToggle(state, action) {
      const q = state.items.find(x => x.id === action.payload.id)
      if(q) q.solved = action.payload.solved
    }
  },
  extraReducers: builder => {
    builder
      .addCase(toggleSolved.rejected, (state, action) => {
        // rollback
        const q = state.items.find(x => x.id === action.payload.id)
        if(q) q.solved = !q.solved
        state.error = action.payload.error
      })
  }
})
export const { optimisticToggle } = questionsSlice.actions
export default questionsSlice.reducer
```
